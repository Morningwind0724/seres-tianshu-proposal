一、方案总览与设计哲学

1.1 一句话定义

「天枢」 = 一个具备自主感知、自主推理、自主决策、自主执行、自主进化能力的设备质量风险管控 AI 数字员工，以"设备状态→质量风险"因果推理为核心，构建"感知—认知—交互—执行—进化"五层闭环。

1.2 设计哲学：三个范式跃迁

范式 传统工业 AI 赛力斯命题要求 「天枢」实现
交互范式 被动问答（人问 AI 答） 主动管控 Agent 7×24 自主监控触发
推理范式 向量 RAG 语义检索 逻辑推理 本体论 SWRL 规则 + Graph RAG
行动范式 输出报告人工处置 自主决策闭环 ReAct Agent 自主编排工具链

1.3 为什么叫「天枢」

天枢是北斗七星第一颗，古代导航基准星。寓意：在赛力斯"AI 定义汽车"转型中，AI 数字员工成为质量管控的导航基准——指引产线在设备退化演变为质量事故前，提前规避风险。

二、四层闭环架构详解

2.1 感知层（Perception）

目标：毫秒级捕获设备状态异常，不漏报不误报。

关键设备与传感器部署：

产线 关键设备 传感器矩阵 采样频率 异常指标
CTP 电池产线 模组堆叠机、激光焊接机 振动加速度计(三轴)、温度热电偶、电流霍尔、视觉相机 振动 10kHz / 温度 1Hz / 视觉 30fps 堆叠精度偏差、焊点气孔率、温度梯度异常

为什么边缘 + 云端双层：
- 边缘侧：毫秒级响应，防止设备故障扩大（如压铸机压力异常 → 100ms 内降速）
- 云端侧：深度推理，跨设备跨工艺的因果分析（如焊装参数偏差 → 总装质量风险）

2.3 交互层（Interaction）

飞书 Aily 智能体三态交互模式：

模式 触发条件 交互形式 示例
主动推送态 Agent 自主发现风险 Aily Bot 推送卡片到责任人飞书 "⚠️ 3号压铸机压力波动超标，预测影响电池壳体气孔率，建议降速10%"
被动问答态 工人主动提问 自然语言问答 工人："扭矩偏差0.3意味着什么？" → Aily 返回带证据链的解释
协同决策态 高风险事件需人工确认 多维表格工单 + 审批流 Agent 生成处置方案 → 工程师审批 → 自动执行

2.4 执行层（Execution）

三级行动权限（工业安全设计）：

风险等级 Agent 自主权 人工介入 执行动作
L1 低风险 全自动 无需 记录入库 + Aily 通知
L2 中风险 半自动 工程师确认 MES 调整排产 + 多维表格工单
L3 高风险 仅建议 必须审批 数字孪生验证 → PLC 降速/停机

数字孪生验证机制（工业安全底线）：
所有影响生产的执行动作（如 PLC 降速/停机），必须先在数字孪生平台模拟验证：
1. Agent 生成执行方案
2. 推送至数字孪生平台，模拟执行后 30 分钟的产线状态
3. 验证不引发次生风险（如降速是否导致后续工位堆料）
4. 验证通过 → 自动执行；验证失败 → 升级人工处置

三、本体论建模完整方案

3.1 为什么必须用本体论（而非纯向量 RAG）

能力 向量 RAG 本体论 + Graph RAG
语义相似度匹配 ✅ ✅
跨实体关系推理 ❌ ✅
因果链推导 ❌ ✅
公理约束检查 ❌ ✅
推理路径可解释 ❌ ✅
处理未见过的组合 ❌（训练数据无则失败） ✅（基于规则推理）

工业场景的硬需求：当 Agent 说"建议降速 10%"时，工程师必须能追溯完整推理链——"3号压铸机振动频率异常 → 关联本体规则：振动>阈值→压射压力波动→铸件气孔风险→质量指标PQ-007超标概率 78%"。向量 RAG 给不出这种因果链。

3.3 完整 OWL 本体定义（核心片段）

<!-- ============================================================
     赛力斯工厂设备-工艺-质量本体（OWL/Turtle 格式）
     工具：Protégé 编辑 + Pellet 推理机
     ============================================================ -->

@prefix : <http://seres.com/ontology/manufacturing#> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix swrl: <http://www.w3.org/2003/11/swrl#> .

# ========== 设备本体 ==========
:Equipment a owl:Class ;
    rdfs:subClassOf [
        a owl:Restriction ;
        owl:onProperty :hasComponent ;
        owl:someValuesFrom :Component
    ] .

:CastingMachine a owl:Class ;
    rdfs:subClassOf :Equipment .

:WeldingRobot a owl:Class ;
    rdfs:subClassOf :Equipment .

:Component a owl:Class .
:HydraulicCylinder a owl:Class ; rdfs:subClassOf :Component .
:Electrode a owl:Class ; rdfs:subClassOf :Component .

# 设备退化等级枚举
:DegradationLevel a owl:Class ;
    owl:oneOf (:Healthy :Warning :Critical :Failed) .

# ========== 工艺本体 ==========
:Process a owl:Class ;
    rdfs:subClassOf [
        a owl:Restriction ;
        owl:onProperty :hasParameter ;
        owl:someValuesFrom :ProcessParameter
    ] .

:DieCasting a owl:Class ; rdfs:subClassOf :Process .
:SpotWelding a owl:Class ; rdfs:subClassOf :Process .

:ProcessParameter a owl:Class .
:CastingPressure a owl:Class ; rdfs:subClassOf :ProcessParameter .
:WeldingCurrent a owl:Class ; rdfs:subClassOf :ProcessParameter .

# ========== 质量本体 ==========
:QualityIndicator a owl:Class .
:Porosity a owl:Class ; rdfs:subClassOf :QualityIndicator .
:Weld Nugget Diameter a owl:Class ; rdfs:subClassOf :QualityIndicator .

# ========== 关系属性 ==========
:hasProcess a owl:ObjectProperty ;
    rdfs:domain :Equipment ;
    rdfs:range :Process .

:affectsQuality a owl:ObjectProperty ;
    rdfs:domain :Process ;
    rdfs:range :QualityIndicator ;
    owl:inverseOf :affectedBy .

:hasDegradationLevel a owl:DatatypeProperty ;
    rdfs:domain :Equipment ;
    rdfs:range xsd:string .

# ========== SWRL 推理规则 ==========

# 规则1：设备退化 → 工艺参数漂移风险
[ a swrl:Imp ;
  swrl:body (
    [ a swrl:ClassAtom ; swrl:classPredicate :Equipment ; swrl:argument1 ?e ]
    [ a swrl:DatavalPropertyAtom ; swrl:propertyPredicate :hasDegradationLevel ;
      swrl:argument1 ?e ; swrl:argument2 "Warning" ]
    [ a swrl:ObjectPropertyAtom ; swrl:propertyPredicate :hasProcess ;
      swrl:argument1 ?e ; swrl:argument2 ?p ]
  ) ;
  swrl:head (
    [ a swrl:BuiltinAtom ; swrl:builtin swrlb:concat ;
      swrl:arguments ("参数漂移风险" ?p) ]
  )
] .

# 规则2：工艺参数漂移 → 质量风险预警
[ a swrl:Imp ;
  swrl:body (
    [ a swrl:ObjectPropertyAtom ; swrl:propertyPredicate :affectsQuality ;
      swrl:argument1 ?p ; swrl:argument2 ?q ]
    [ a swrl:BuiltinAtom ; swrl:builtin :parameterDrift ;
      swrl:arguments (?p ?drift) ]
    [ a swrl:BuiltinAtom ; swrl:builtin swrlb:greaterThan ;
      swrl:arguments (?drift 0.05) ]
  ) ;
  swrl:head (
    [ a swrl:ClassAtom ; swrl:classPredicate :QualityRisk ; swrl:argument1 ?q ]
  )
] .

3.4 本体论实例化（赛力斯真实设备）

# 3号压铸机实例
:Equip_LK9000T_003 a :CastingMachine ;
    :model "力劲 9000T 一体化压铸机" ;
    :location "赛力斯超级工厂 A区" ;
    :hasComponent :HydraulicCylinder_003A, :Furnace_003B ;
    :hasProcess :Proc_DieCasting_003 ;
    :hasDegradationLevel "Warning" ;  # 当前退化等级
    :lastMaintenance "2026-06-15" .

# 3号压铸机的压铸工艺
:Proc_DieCasting_003 a :DieCasting ;
    :hasParameter :Param_Pressure_003, :Param_Speed_003, :Param_Temp_003 ;
    :affectsQuality :QI_Porosity, :QI_Shrinkage .

# 质量指标实例
:QI_Porosity a :Porosity ;
    :standardRange "≤ 0.5%" ;
    :currentValue "0.8%" ;  # 超标
    :detectionMethod "X光检测" .

4.3 Graph RAG 核心代码

"""
天枢 Graph RAG 引擎
技术栈: Neo4j + LlamaIndex GraphRAG + OpenAI Embeddings
"""
from neo4j import GraphDatabase
from llama_index.graph_stores.neo4j import Neo4jGraphStore
from llama_index.embeddings.openai import OpenAIEmbedding
from llama_index.llms.deepseek import DeepSeekLLM

class TianshuGraphRAG:
    def __init__(self, neo4j_uri, neo4j_user, neo4j_pwd):
        # 1. 连接知识图谱
        self.driver = GraphDatabase.driver(
            neo4j_uri, auth=(neo4j_user, neo4j_pwd)
        )
        # 2. 初始化嵌入模型
        self.embed_model = OpenAIEmbedding(model="text-embedding-3-large")
        # 3. 初始化 LLM(赛力斯已与 DeepSeek 合作)
        self.llm = DeepSeekLLM(model="deepseek-v3")
        # 4. 加载本体推理机
        self.reasoner = OntologyReasoner("seres_ontology.owl")

    def retrieve(self, anomaly_signal):
        """三阶段检索"""
        # Stage 1: 本体推理缩范围
        candidate_causes = self.reasoner.infer_causes(anomaly_signal)
        # 示例输出: ["液压缸退化", "模具温度异常", "压射速度漂移"]

        # Stage 2: Graph RAG 深度检索
        subgraphs = []
        with self.driver.session() as session:
            for cause in candidate_causes:
                # Cypher 查询: 3 跳邻域子图
                cypher = """
                MATCH path = (c:Component {name: $cause})-[*1..3]-(related)
                WHERE ALL(r IN relationships(path)
                          WHERE type(r) IN ['affects', 'causes',
                                            'correlates_with', 'mitigates_by'])
                RETURN path, nodes(path) as nodes, relationships(path) as rels
                LIMIT 50
                """
                result = session.run(cypher, cause=cause)
                subgraphs.append(self._build_subgraph(result))

        # Stage 3: LLM 生成证据链
        evidence_chain = self._generate_evidence(
            anomaly_signal, candidate_causes, subgraphs
        )
        return evidence_chain

    def _generate_evidence(self, anomaly, causes, subgraphs):
        """LLM 生成可解释证据链"""
        prompt = f"""
        你是赛力斯超级工厂的设备质量专家。
        异常信号: {anomaly}
        候选根因: {causes}
        知识图谱子图: {self._subgraph_to_text(subgraphs)}

        请输出 JSON:
        {{
            "root_cause": "最可能的根因",
            "confidence": 0.0-1.0,
            "evidence_chain": ["证据1", "证据2", ...],
            "affected_quality": "受影响的质量指标",
            "action_plan": "处置建议",
            "reference_cases": ["历史案例ID"]
        }}
        """
        response = self.llm.complete(prompt)
        return self._parse_json(response)

    def _subgraph_to_text(self, subgraphs):
        """将子图序列化为 LLM 可读文本"""
        text_parts = []
        for sg in subgraphs:
            for node in sg.nodes:
                text_parts.append(f"节点: {node.label}({node.type})")
            for edge in sg.edges:
                text_parts.append(
                    f"关系: {edge.source} -[{edge.type}]-> {edge.target}"
                )
        return "\n".join(text_parts)

5.2 ReAct 框架实现

"""
天枢多智能体 ReAct 实现
框架: LangGraph
"""
from langgraph.graph import StateGraph, END
from typing import TypedDict, List, Dict

class AgentState(TypedDict):
    anomaly: Dict                    # 异常信号
    device_health: Dict              # 设备健康Agent输出
    quality_risk: Dict               # 质量风险Agent输出
    decision: Dict                   # 决策Agent输出
    messages: List[str]              # 协同消息记录

class DeviceHealthAgent:
    """设备健康Agent: 诊断设备退化根因"""
    def __init__(self, llm, tools):
        self.llm = llm
        self.tools = tools  # 振动分析、温度趋势、维保历史

    def __call__(self, state: AgentState) -> AgentState:
        # ReAct: Reason + Act 循环
        thought = self.llm.invoke(
            f"异常: {state['anomaly']}\n"
            f"思考: 需要诊断设备退化根因"
        )
        # 调用工具
        vibration = self.tools["vibration_analyzer"](
            state["anomaly"]["device_id"]
        )
        maintenance = self.tools["maintenance_history"](
            state["anomaly"]["device_id"]
        )
        # 推理根因
        diagnosis = self.llm.invoke(
            f"振动数据: {vibration}\n维保历史: {maintenance}\n"
            f"输出设备健康度评分(0-1)和退化根因"
        )
        state["device_health"] = self._parse(diagnosis)
        state["messages"].append(
            f"[DeviceHealthAgent] {state['device_health']}"
        )
        return state

class QualityRiskAgent:
    """质量风险Agent: 预判质量影响"""
    def __call__(self, state: AgentState) -> AgentState:
        # 基于设备退化, 预判质量风险
        risk = self.graph_rag.retrieve(
            anomaly=state["anomaly"],
            device_health=state["device_health"]
        )
        state["quality_risk"] = risk
        state["messages"].append(
            f"[QualityRiskAgent] {risk}"
        )
        return state

class DecisionAgent:
    """决策Agent: 综合决策与执行编排"""
    def __call__(self, state: AgentState) -> AgentState:
        # 综合设备健康 + 质量风险, 生成处置方案
        plan = self.llm.invoke(
            f"设备健康: {state['device_health']}\n"
            f"质量风险: {state['quality_risk']}\n"
            f"输出处置方案(风险等级/执行动作/责任人)"
        )
        # 数字孪生验证(高风险必须)
        if self._risk_level(plan) == "L3":
            simulation = self.tools["digital_twin"].simulate(plan)
            if not simulation["safe"]:
                plan = self._fallback_plan(plan)
        # 飞书Aily推送
        self.tools["feishu_aily"].push(
            to=plan["responsible"],
            message=plan
        )
        state["decision"] = plan
        return state

# 构建 LangGraph 工作流
workflow = StateGraph(AgentState)
workflow.add_node("device_health", DeviceHealthAgent())
workflow.add_node("quality_risk", QualityRiskAgent())
workflow.add_node("decision", DecisionAgent())
workflow.add_edge("device_health", "quality_risk")
workflow.add_edge("quality_risk", "decision")
workflow.add_edge("decision", END)
workflow.set_entry_point("device_health")
app = workflow.compile()

7.2 SPC 控制限设计示例

"""
SPC 控制限检测 - 压铸机压力参数
基于历史数据计算控制限, 实时检测异常
"""
import numpy as np

class SPCMonitor:
    def __init__(self, historical_data, param_name):
        self.param = param_name
        # 基于历史数据计算控制限
        self.mean = np.mean(historical_data)
        self.std = np.std(historical_data)
        self.ucl = self.mean + 3 * self.std  # 上控制限
        self.lcl = self.mean - 3 * self.std  # 下控制限
        self.uwl = self.mean + 2 * self.std  # 上警告限
        self.lwl = self.mean - 2 * self.std  # 下警告限

    def check(self, value):
        """返回异常等级"""
        if value > self.ucl or value < self.lcl:
            return "CRITICAL"  # 超控制限, 立即触发
        elif value > self.uwl or value < self.lwl:
            return "WARNING"   # 超警告限, 预警
        return "NORMAL"

# 示例: 3号压铸机压射压力
pressure_history = load_history("Equip_LK9000T_003", "CastingPressure")

        })

12.2 痛点验证逻辑

用户反馈的"低频共振售后处理仍未消除"，本质是传统AI单点检测做不到跨实体根因推理——共振问题涉及车身结构、装配工艺、材料批次多个维度，向量检索无法建立因果链。「天枢」的本体论+Graph RAG正是为此设计：通过本体规则将"车身振动异常→装配扭矩偏差→结构共振"的隐性因果链显式化，让Agent能像资深工程师一样"顺藤摸瓜"定位根因。

16.2 降级策略（Graceful Degradation）

正常模式: 全自动 Agent 闭环
    ↓ (LLM服务不可用)
降级模式1: 本体推理+规则引擎(无LLM)
    ↓ (知识图谱不可用)

    ↓ (边缘计算不可用)

十八、差异化加分点总结

18.1 五大差异化卖点

  1. 范式哲学跃迁：从"被动问答"到"主动发现"——这不是技术升级，是产品哲学的跃迁。赛力斯命题原文明确说"传统工业知识图谱大多停留在被动问答层面"，我们的方案精准回应这一痛点。

  2. 可解释性是工业信任前提：本体论SWRL规则推理输出完整证据链，让"AI说了算"变为"AI说了为什么"——这是工业场景采纳AI的硬性门槛，向量RAG黑盒给不了。

  3. 飞书非典型工业应用：把办公协作的Aily改造为产线"虚拟质量工程师"，多维表格替代传统MES工单系统——跨场景映射能力正是评审维度"AI应用创新性"要的分。

  4. 真实痛点锚定：M7低频共振、智驾误触发等用户反馈，证明方案回应真实需求而非空中楼阁——评委最容易记住"有数据支撑的痛点"。

  5. 三期推广路径：不是一次性Demo，是可泛化至全行业的架构——本体+KG+Agent架构可平滑迁移至其他整车厂和离散制造场景，呼应"可推广性"评分维度。

18.2 一句话总结

「天枢」不是又一个AI质检工具，是赛力斯从"AI工具"迈向"AI组织"的关键支点——用本体论建模设备-工艺-质量因果链，让Agent具备逻辑推理能力，在设备退化演变为质量事故前，提前规避风险。这是"AI定义汽车"时代，质量管控的范式重构。
